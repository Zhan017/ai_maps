# VOYGR Replica

A "better maps API" demo: a FastAPI service that validates, enriches, and
serves canonical place data, fronted by a Leaflet map + chat UI. Modelled
on VOYGR's place-intelligence product. See `docs/voygr_replica.md` for the
full spec.

The stack is **Postgres + PostGIS + pgvector** with raw SQL via `psycopg`,
**OpenAI** for the embeddings + chat, and **FastAPI** for the API.

> All "enrichment data" — hours, amenities, sources, vibe attributes — is
> generated by the seed script. There is no real crawler. The pipeline shape
> is real; the data is mock.

## Quick start

```bash
docker compose up -d --build db          # postgis + pgvector
pip install -r requirements.txt

python -m scripts.seed                    # ~30s; prints two API keys

# optional but recommended — semantic search needs embeddings
OPENAI_API_KEY=sk-... python -m scripts.build_embeddings

OPENAI_API_KEY=sk-... uvicorn app.main:app --reload
```

Open <http://localhost:8000/>.

## Endpoint cheat-sheet

All `/v1/*` and `/internal/*` routes require `X-API-Key`.

| Endpoint | Purpose |
|---|---|
| `POST /v1/places:validate-enrich` | Match customer places to canonical IDs and return enriched profiles |
| `GET /v1/places/{id}` | Full profile (hours, amenities, attributes; optional sources + history) |
| `POST /v1/places:status` | Batch status + freshness for a list of canonical IDs |
| `POST /v1/places/{id}:feedback` | Submit a correction (`closed`, `wrong_address`, etc.) |
| `GET /v1/places:search` | Hybrid geo + semantic search; returns `reasons[]` and `freshness{}` |
| `GET /internal/places/{id}/debug` | Profile + customer refs + feedback |
| `POST /internal/places/{id}/override` | Manual status / name overrides |
| `GET /api/nearby`, `POST /api/chat`, `DELETE /api/chat/{sid}`, `GET /api/health` | Legacy unauthenticated routes used by the demo UI |

## Examples

```bash
KEY=vk_xxx  # from scripts.seed output

curl -s "localhost:8000/v1/places:search?lat=51.13&lng=71.43&radius_m=1500&category=cafe&open_now=true&wifi=true&limit=5" \
  -H "X-API-Key: $KEY" | jq

curl -s -X POST localhost:8000/v1/places:validate-enrich \
  -H "X-API-Key: $KEY" -H "Content-Type: application/json" \
  -d '{"places":[{"customer_place_id":"abc","name":"Baiterek Cafe","lat":51.13,"lng":71.43,"category":"cafe"}]}' | jq

# Natural-language search via the chat endpoint (no API key)
curl -s -X POST localhost:8000/api/chat -H "Content-Type: application/json" \
  -d '{"session_id":"test","message":"quiet coffee with wifi near here","map_center":{"lat":51.13,"lon":71.43}}' | jq
```

## Tests

```bash
pytest                                    # requires seeded DB
```

The semantic test auto-skips unless `OPENAI_API_KEY` is set *and*
`places_vectors` has been populated.

## Layout

```
app/
  main.py              FastAPI entrypoint
  api/v1/              places, search, admin, legacy routers
  core/                config, security
  db/                  schema.sql, pool, raw SQL helpers
  services/            matching, validation, enrichment, search, embeddings
  utils/               geo, text
db/Dockerfile          postgis + pgvector
scripts/
  seed.py              50k Astana places + mock enrichment
  build_embeddings.py  OpenAI embeddings → places_vectors
static/index.html      Leaflet UI + chat
tests/
```
